## 3. 主要テストケースの詳細手順

### 3.1 出勤打刻（PC / スマートフォン）
1. システムに個人 ID とパスワードでログインする。
2. ダッシュボード画面中央の「出勤」ボタンを確認する。
3. ボタンをクリック（タップ）する。
4. 「出勤を記録しました」というメッセージが表示されることを確認する。
5. 打刻時刻が現在時刻と一致していることを、勤務表画面で目視確認する。

### 3.2 有給休暇申請・承認
1. 従業員アカウントでログインし、サイドメニューの「休暇申請」をクリックする。
2. 休暇種別に「年次有給（全日）」を選択する。
3. 取得日をカレンダーから選択し、申請理由を入力して「申請」ボタンをクリックする。
4. 管理職アカウントでログインし、「承認待ち一覧」から該当の申請を確認する。
5. 「承認」ボタンをクリックし、ステータスが「完了」になることを確認する。
6. 再度従業員アカウントでログインし、有給残数が正しく 1日分減っていることを確認する。

### 3.3 36協定アラートの確認
1. 管理者アカウントで、特定のテスト用従業員の法定外残業時間を 35時間 59分に設定する。
2. 該当従業員でログインし、2分間の勤務打刻（または修正申請）を行う。
3. 月間残業合計が 36時間（45時間の80%）を超えたことを確認する。
4. システムから本人宛および設定された上長宛に、アラート通知（メールまたはプッシュ通知）が届くことを確認する。

## 4. 業務シナリオテスト

### シナリオA: 一般従業員の1ヶ月の勤怠管理
1. **日次打刻**: 毎朝 PC ブラウザから出勤を打刻し、夕方に退勤を打刻する。
2. **リモートワーク**: 週に1回、在宅勤務時にスマホから打刻し、位置情報が記録されていることを確認する。
3. **休暇申請**: 月中に 1日分の有給休暇を申請し、承認される。
4. **月末確認**: 月末に勤務表画面を開き、有給取得日が「休暇」として集計され、総労働時間が正しく表示されていることを確認する。
5. **アラート**: 残業が 36時間を超えた際にアラートを受け取り、それ以上の残業を抑制する行動をとる。

### シナリオB: 管理職の部門管理
1. **申請承認**: 月末に部下から届いた「打刻修正申請」3件と「休暇申請」2件を、内容を確認して一括承認する。
2. **状況把握**: 部門勤務状況画面を開き、メンバーの残業時間の進捗（36協定閾値への接近）をグラフで視覚的に把握する。
3. **是正指示**: アラートが出ているメンバーを特定し、業務調整を行う。

### シナリオC: 総務部の月次処理
1. **マスタ更新**: 新入社員の情報を従業員マスタに登録し、初期有給（10日）を付与する。
2. **全社集計**: 月初に前月分の全社勤怠データを確定・集計する。
3. **CSV連携**: 「給与大臣」用の連携 CSV を出力し、指定の項目順（社員コード、氏名、平日残業、休日労働、深夜労働等）で出力されていることを確認する。
4. **コンプライアンス確認**: 「有給年5日取得義務」の進捗レポートを確認し、未達見込み者への催促リストを作成する。

## 5. 受け入れ基準

- **全テストケースの合格率**: 100%（実施した全ての UAT ケースが期待結果を満たすこと）
- **致命的な不具合 (Critical)**: 0件（システム停止、データ破損、セキュリティ不備等）
- **主要な不具合 (Major)**: 3件以下（主要機能は動作するが一部不便な点がある。運用回避策が存在すること）
- **軽微な不具合 (Minor)**: 協議の上、修正時期を決定。本番稼働の妨げにならないこと
