# Gemini CLI で実践する V字モデル開発 ハンズオン

## 従業員勤怠管理システム（Employee Attendance Management System）

本ハンズオンでは、**Gemini CLI** を使って V字モデルの全10工程を一気通貫で体験します。
題材は中堅企業向けの「従業員勤怠管理システム」です。

---

## ハンズオンの全体像

```
所要時間: 約 3〜4 時間（各工程 15〜25 分）
前提知識: プログラミング基礎（Python）、Web API の基本概念
成果物:   V字モデル全10工程の設計書 + 動作するコード + テスト一式
```

### V字モデルの流れとハンズオンの対応

```
Step 01 要件分析 ─────────────────────────────── Step 10 受け入れテスト
  │       RFP の要求 ←→ 受け入れ検証               │
  Step 02 要件定義 ───────────────────── Step 09 システムテスト
    │       非機能要件 ←→ 性能・セキュリティ検証     │
    Step 03 基本設計 ─────────────── Step 08 結合テスト
      │       API 設計 ←→ API 連携検証          │
      Step 04 詳細設計 ─────── Step 07 単体テスト
        │       クラス設計 ←→ メソッド検証    │
        Step 05 コーディング ── Step 06 コードレビュー
```

**核心**: 左側で「決めたこと」を、右側の対応するテストで「検証」する。
AI はこの対応関係を自動的にトレースし、漏れを防ぎます。

## 題材: 従業員勤怠管理システム

### なぜこの題材なのか

1. **誰でも理解できる** -- 勤怠管理は全てのビジネスパーソンが日常的に接する業務
2. **明確なビジネスルール** -- 労働基準法・36協定という法的根拠に基づく計算ロジック
3. **適度な複雑さ** -- CRUD + 計算ロジック + アラート + レポートの組み合わせ
4. **テストしやすい** -- 残業時間計算や有給残日数など、入力と期待値が明確

### システムで解決する課題

| # | 現状の課題 | システム化で実現すること |
|---|-----------|----------------------|
| 1 | Excel での勤怠集計に毎月3日かかる | 自動集計でリアルタイム確認 |
| 2 | 36協定の超過を月末まで気づけない | 残業時間の閾値超過で自動アラート |
| 3 | 有給残日数の問い合わせが総務に集中 | 従業員がセルフサービスで確認 |
| 4 | 紙の申請書で承認に2〜3日かかる | ワークフローで即時承認 |

## 進め方

### AI 駆動の自律開発フロー

各 Step では、**1つのプロンプト** を Gemini CLI に入力するだけで、AI が以下のサイクルを自律的に実行します:

```
┌─────────────────────────────────────────────┐
│  Phase 1: 成果物の生成と保存                    │
│    → プロンプトの仕様に従い成果物を生成           │
│    → 指定ファイルに保存                         │
├─────────────────────────────────────────────┤
│  Phase 2: セルフレビューと修正                   │
│    → 確認項目を1つずつチェック                   │
│    → 問題があれば修正して再保存                   │
│    → 全項目 ✅ になるまで繰り返す                │
├─────────────────────────────────────────────┤
│  Phase 3: 完了報告                             │
│    → チェック結果一覧を出力                      │
│    → 修正箇所のサマリーを報告                    │
└─────────────────────────────────────────────┘
```

**人間の役割**: プロンプトを入力し、AI の完了報告を確認する。必要に応じて追加の修正を依頼する。

### Gemini CLI の使い方

```bash
# 対話モードで起動（推奨）
cd hands-on/01_requirements_analysis
gemini

# 対話モード中にプロンプトを貼り付けて実行
# AI が自律的に生成・保存・レビュー・修正を行う
```

### 前工程の成果物が次の入力になる

```
rfp.md  →  [Step 01]  →  ai_analysis.md（参考）
rfp.md  →  [Step 02]  →  requirements.md
                              ↓
                         [Step 03]  →  basic_design.md
                                          ↓
                                     [Step 04]  →  detailed_design.md
                                                      ↓
                                                 [Step 05]  →  API + テスト + コード
```

各プロンプトの末尾にある `@ファイルパス` で前工程の成果物を参照しています。

## 注意事項

- Gemini CLI の出力は **毎回異なります**。本ハンズオンに記載の「期待される出力」は参考例です
- 各工程のプロンプトは **1回の入力で自律実行** されます。AI が生成・保存・レビュー・修正を一連で行います
- 各工程の成果物は、次の工程の **入力** になります。AI の完了報告を確認してから次に進んでください
- Step 05（コーディング）では **API 層を含む完全な実装** が生成され、テストも自動実行されます
- AI がセルフレビューで問題を検出・修正するプロセスを観察することで、AI 駆動開発の実践力が身につきます
