## 作業手順

1. 要件定義書の機能要件をすべて読み、ドメインモデル（エンティティ）を抽出する
2. ER 図を Mermaid 記法で作成する
3. API エンドポイント一覧を RESTful に設計する
4. 画面遷移図を Mermaid 記法で作成する
5. システム構成図を記述する

## 出力形式

### 1. システム構成図

クライアント → API Gateway → FastAPI → PostgreSQL のような構成を詳細に記述する。

### 2. ER図（Mermaid 記法）

以下のエンティティを含めること:

- employees: 従業員（氏名、部門、就業形態、入社日）
- departments: 部門
- attendance_records: 打刻記録（出勤/退勤/休憩開始/休憩終了）
- daily_summaries: 日次勤怠集計（勤務時間、残業時間、深夜時間）
- monthly_summaries: 月次勤怠集計
- leave_balances: 有給残日数
- leave_requests: 休暇申請
- holidays: 祝日・会社カレンダー
- alerts: アラート履歴

各テーブルに以下を含めること:
- 主キー（UUID）
- 外部キー
- 主要カラム（型を含む）
- created_at, updated_at

### 3. API エンドポイント一覧

以下の形式で全 API を設計:

| # | メソッド | パス | 説明 | リクエスト | レスポンス | 認可 |
|---|---------|------|------|-----------|-----------|------|

API 設計のルール:
- RESTful 設計に従う（リソース指向の URL）
- バージョニング: /api/v1/ プレフィックス
- 認証: Bearer Token（JWT）
- ページネーション: limit/offset パラメータ
- エラーレスポンス: RFC 7807 (Problem Details) 形式

最低 15 本以上の API を設計すること。

### 4. 画面一覧と画面遷移図

画面一覧を表で記述し、画面遷移を Mermaid の stateDiagram で記述する。

### 5. 認証・認可設計

ロールベースアクセス制御 (RBAC) の設計:

| ロール | 権限 |
|--------|------|

## 技術スタック

- Backend: Python 3.12+ / FastAPI
- DB: PostgreSQL 15+
- ORM: SQLAlchemy 2.0
- 認証: JWT (PyJWT)
- バリデーション: Pydantic v2
- テスト: pytest
- API ドキュメント: OpenAPI (FastAPI 自動生成)

## やってはいけないこと

- テーブル名にキャメルケースを使わないこと（snake_case 統一）
- API パスに動詞を使わないこと（RESTful に名詞で設計）
- 1つの API に複数の責務を持たせないこと
- ER 図で外部キーの関連を省略しないこと

## 品質チェックリスト

- [ ] ロール別のアクセス制御が全 API に定義されているか
- [ ] 打刻 → 日次集計 → 月次集計のデータフローが明確か

## この工程で学ぶこと

| 学習ポイント | 説明 |
|------------|------|
| Mermaid 記法の活用 | ER図や画面遷移図を Mermaid で出力させることで、レビューしやすい図が得られる |
| 技術スタックの明示 | 使用技術を指定することで、その技術に最適化された設計が得られる |
| API 設計のルール指定 | RESTful、バージョニング、エラー形式などのルールを指定する |
| 前工程との対応 | 要件定義の機能ID（F-01〜）が全て API に対応していることを確認する |
