# 従業員勤怠管理システム 要件定義書

## 1. 用語定義

システムで使用する用語を以下の通り定義する。

| 用語 | 定義 | 備考 |
|------|------|------|
| 法定労働時間 | 労働基準法で定められた労働時間の上限（原則として1日8時間、1週40時間）。 | |
| 所定労働時間 | 就業規則等で定められた、各従業員が働くべき時間。 | 雇用契約により異なる。 |
| 法定内残業 | 所定労働時間を超え、かつ法定労働時間に達するまでの労働時間。 | 割増賃金の支払義務はないが、社内規定により算出対象とする。 |
| 法定外残業 | 法定労働時間（1日8時間または1週40時間）を超えた労働時間。 | 25%以上の割増賃金の対象。 |
| 深夜勤務 | 22:00から翌5:00までの時間帯における労働。 | 25%以上の割増賃金の対象。 |
| 36協定 | 労働基準法第36条に基づく労使協定。時間外・休日労働の限度時間を定める。 | |
| フレックスタイム制 | 1ヶ月以内の清算期間内の総労働時間をあらかじめ定め、その範囲内で日々の始業・終業時刻を従業員が決定する制度。 | |
| コアタイム | フレックスタイム制において、労働しなければならない時間帯（10:00〜15:00）。 | |
| 清算期間 | フレックスタイム制において労働時間の過不足を判定する期間（本システムでは1ヶ月）。 | |
| 時間単位有給 | 有給休暇を1時間単位で取得できる制度。 | 年間5日（40時間）が上限。 |

## 2. ビジネスルール

労働基準法および社内規定に基づき、以下の計算ルールを適用する。

### 2.1 労働時間の計算
- **単位**: 1分単位で集計する（切り捨て・切り上げ等の丸め処理は行わない）。
- **法定労働時間**: 1日8時間、週40時間とする。
- **深夜勤務**: 22:00〜翌5:00の間の勤務時間を集計する。

### 2.2 休憩時間の自動控除
実労働時間（拘束時間から手動入力の休憩を除いた時間）に基づき、以下の通り自動控除する（労基法34条準拠）。
- 実労働時間が6時間を超える場合：45分を控除
- 実労働時間が8時間を超える場合：60分を控除

### 2.3 残業時間の集計
- **法定内残業**: 実労働時間が所定労働時間を超え、かつ8時間に達するまでの時間。
- **法定外残業**: 実労働時間が8時間を超えた時間、または週の合計労働時間が40時間を超えた時間（重複は避ける）。
- **36協定上限**:
    - 月間：45時間（法定外残業＋法定休日労働）
    - 年間：360時間（法定外残業）

### 2.4 有給休暇の管理
- **付与日数（勤続年数別）**:
    | 勤続年数 | 0.5年 | 1.5年 | 2.5年 | 3.5年 | 4.5年 | 5.5年 | 6.5年以上 |
    |----------|-------|-------|-------|-------|-------|-------|-----------|
    | 付与日数 | 10日 | 11日 | 12日 | 14日 | 16日 | 18日 | 20日 |
- **取得義務**: 年10日以上の有給が付与される従業員に対し、付与日から1年以内に5日取得させることを必須とする。
- **時間単位有給**: 年間5日分（40時間）を上限とし、1時間単位で取得可能とする。

### 2.5 フレックスタイム制
- **清算期間**: 1ヶ月（毎月1日〜末日）。
- **所定労働時間（標準）**: 清算期間内の「所定勤務日数 × 8時間」を標準とし、実労働時間との差分を過不足時間として算出する。

## 3. 機能要件一覧

| 要件ID | カテゴリ | 要件名 | 要件詳細 | 受け入れ基準 | 優先度 |
|--------|---------|--------|---------|-------------|--------|
| F-01 | 打刻 | マルチデバイス打刻 | PCブラウザおよびスマートフォンから、出勤・退勤の打刻ができること。 | PC/スマホの両端末で打刻操作が可能で、サーバーに正確な時刻が記録されること。 | 高 |
| F-02 | 打刻 | 位置情報記録 | 打刻実行時にGPS情報を取得し、緯度・経度情報を時刻と共に保存する。 | 管理画面から、打刻時の地図上のピンまたは緯度経度情報が確認できること。 | 中 |
| F-03 | 打刻 | 修正申請 | 打刻忘れや誤打刻に対し、翌日23:59までに理由を添えて修正申請ができる。 | 申請が上長に通知され、承認されると勤務データが更新・履歴保存されること。 | 高 |
| F-04 | 集計 | 日次/月次自動集計 | 打刻データを基に、日次・週次・月次の合計労働時間、欠勤日数を算出する。 | 1分単位で計算され、月次サマリー画面で各集計項目が正しく表示されること。 | 高 |
| F-05 | 集計 | 残業・深夜計算 | ビジネスルール2.3に基づき、法定内/外残業および深夜勤務時間を個別に算出する。 | テストデータ（8h超、22時以降勤務等）を入力し、集計値が期待値と一致すること。 | 高 |
| F-06 | 集計 | フレックス過不足計算 | 清算期間終了時に、標準労働時間に対する実労働時間の過不足時間を算出する。 | 月次集計結果に、プラスまたはマイナスの時間数が正確に表示されること。 | 高 |
| F-07 | 休暇 | 有給残数管理 | 入社日を起点とした自動付与、および申請承認による残日数の減算を行う。 | 付与日および有効期限（2年）が管理され、失効処理が自動で行われること。 | 高 |
| F-08 | 休暇 | 取得義務状況管理 | 年5日の取得義務に対する「取得済」「未取得」の状況を個別に可視化する。 | 管理画面で、取得義務対象者ごとの残り取得必要日数が確認できること。 | 高 |
| F-09 | 休暇 | 休暇申請ワークフロー | 全日・半日・時間単位の有給休暇申請および承認・却下フローを提供する。 | 申請が承認された際、即座に有給残数および勤務表に反映されること。 | 高 |
| F-10 | 通知 | 36協定アラート | 月の法定外残業が36時間（45時間の80%）に到達した際、本人と上長に通知する。 | 設定した閾値（36h）を超えたタイミングで即座にアラートメール等が飛ぶこと。 | 高 |
| F-11 | 通知 | 打刻漏れ通知 | 退勤予定時刻を過ぎて打刻がない場合、当日中に本人へプッシュ通知を行う。 | 打刻漏れ対象者のみに、設定した時刻（例：19:00）に通知が届くこと。 | 中 |
| F-12 | 出力 | 給与大臣連携CSV出力 | 指定されたフォーマット（項目順、型）で給与計算ソフト用のCSVを出力する。 | 出力されたCSVを給与大臣のインポート機能でエラーなく読み込めること。 | 高 |
| F-13 | 分析 | 残業推移グラフ表示 | 部門別・個人別の残業時間の推移を6ヶ月分、グラフ形式で表示する。 | ダッシュボード上で6ヶ月分の推移が棒グラフまたは折れ線グラフで表示されること。 | 中 |
| F-14 | 管理 | マスタ管理 | 従業員、部門、祝日、会社カレンダーの登録・更新・削除ができる。 | 祝日設定が、特定日の休日判定（残業計算等）に正しく反映されること。 | 高 |

## 4. 非機能要件一覧

| 要件ID | カテゴリ | 要件名 | 要件詳細 | 測定基準 |
|--------|---------|--------|---------|---------|
| NF-01 | 可用性 | 稼働時間 | システム全体は平日7:00〜23:00稼働。ただし、打刻機能は24時間稼働とする。 | 月間稼働率 99.9% 以上。 |
| NF-02 | 性能 | 応答時間 | 打刻および画面遷移のレスポンス時間を一定以下に保つ。 | 通常時の画面レスポンスが 2秒以内 であること。 |
| NF-03 | 保守性 | データ保持期間 | 勤怠記録および申請データを法定期間に基づき保存する。 | 過去 5年間 のデータを欠損なく保持し、照会可能であること。 |
| NF-04 | 安全性 | アクセス制御 | RFP第4項で定義された4つの権限ロール（一般、管理職、総務、管理者）に基づき、閲覧・操作制限を行う。 | 他者の個人情報への不正アクセス試行が全て遮断されること。 | 高 |
| NF-05 | 運用 | バックアップ | 万が一のデータ消失に備え、定期的なバックアップを取得する。 | 毎日1回 のフルバックアップを別リージョンに保存すること。 |
