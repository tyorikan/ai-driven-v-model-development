```
        詳細設計 ──────── 単体テスト

```

### 2. 以下のプロンプトを入力

````
あなたは Python (FastAPI / SQLAlchemy) のシニアコードレビュアーです。
セキュリティ、パフォーマンス、保守性に特に詳しいです。

以下のコードファイルをレビューし、問題を修正してください。

## 以下の 3 つのフェーズを順番に自律的に実行してください

---

### Phase 1: コードレビューの実施と結果保存

以下の 7 つの観点で全てのコードファイルをレビューし、
レビュー結果を `../06_code_review/review_report.md` に保存してください。

#### レビュー観点（7つ全てを必ず実施すること）

1. **仕様整合性**
   - 勤務時間の計算ロジックが正しいか
   - 休憩時間の自動控除が労基法34条に準拠しているか（6h超→45分、8h超→60分）

2. **セキュリティ**
   - SQL インジェクションの危険性
   - 個人情報（位置情報含む）の取り扱い

3. **パフォーマンス**
   - N+1 問題
   - 不要なクエリ

4. **エラーハンドリング**
   - 例外の握りつぶし（except: pass）
   - トランザクション制御の不備

5. **型安全性**
   - 型ヒントの欠落
   - Any 型の乱用

6. **命名規約**
   - 命名の一貫性（snake_case）

7. **アーキテクチャ**
   - Controller 層にビジネスロジックが漏れていないか
   - サービス層の責務分離

#### 出力形式

##### 指摘一覧

| # | ファイル | 行番号 | 深刻度 | カテゴリ | 指摘内容 | 修正案 |
|---|---------|--------|--------|---------|---------|--------|

深刻度の定義:
- **Critical**: セキュリティ脆弱性、データ破損の可能性 → 必ず修正
- **Major**: パフォーマンス問題、ロジックバグ → 修正推奨
- **Minor**: 命名規約違反、コードスタイル → 修正任意

##### サマリー

- Critical: X件
- Major: X件
- Minor: X件
- 総合評価: （合格 / 条件付き合格 / 要修正）

---

### Phase 2: 指摘事項の修正とテスト

Critical と Major の指摘を全て修正してください:

1. 各指摘に対応するコードを修正し、修正箇所にコメント `# REVIEW FIX: 指摘内容` を付ける
2. 修正したファイルを保存する
3. テストを実行して全て PASS することを確認する:

```bash
python -m pytest tests/ -v
```

4. テストが失敗した場合は修正して再テスト（全 PASS するまで繰り返す）
5. `../06_code_review/review_report.md` に修正結果を追記する

---

### Phase 3: 完了報告

全ての修正が完了し、テストが全 PASS したら、以下の形式で完了報告を出力してください:

```
## 完了報告

### レビュー結果サマリー
- Critical: X件（全て修正済み）
- Major: X件（全て修正済み）
- Minor: X件
- 総合評価: 合格

### テスト結果
- 全テスト: XX passed, 0 failed

### 修正したファイル一覧
- ファイル名: 修正内容の概要
...

### 成果物
- レビューレポート: ../06_code_review/review_report.md
```

## やってはいけないこと

- 全て「問題なし」で終わらせないこと（最低 5 件は指摘すること）
- 修正案なしの指摘をしないこと（必ず具体的な修正コードを示す）
- 深刻度を全て Minor にしないこと
- 修正後にテストを実行しないこと（必ず PASS を確認する）

@app/services/attendance_service.py
@app/models/attendance.py
@app/models/employee.py
@app/core/exceptions.py
@app/schemas/attendance.py
@app/api/v1/attendance.py
@app/main.py
@app/db/session.py
````

## この工程で学ぶこと

| 学習ポイント | 説明 |
|------------|------|
| AI レビューは網羅的 | 7つの観点を漏れなくチェックでき、人間のレビューの補完になる |
| 自律的な修正サイクル | レビュー → 修正 → テスト → 確認を AI が一連で実行する |
| 修正案の具体性 | 「こうすべき」だけでなく「こう書き換える」という具体的なコードを出力させる |
| AI レビューの限界 | ビジネスロジックの妥当性（労基法の正確性など）は人間の判断が必要 |
