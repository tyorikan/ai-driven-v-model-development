### 1.2 負荷テストシナリオ

```python
# Locust による負荷テストスクリプト（AI 生成例）

from locust import HttpUser, task, between


class QualitySystemUser(HttpUser):
    """品質管理システムの典型的なユーザー操作を再現する。"""

    wait_time = between(1, 3)

    @task(5)
    def view_inspections(self):
        """検査結果一覧の閲覧（最も頻度が高い操作）。"""
        self.client.get("/api/v1/inspections?limit=50")

    @task(3)
    def create_inspection(self):
        """検査結果の登録（検査員の操作）。"""
        self.client.post("/api/v1/inspections", json={
            "lot_number": "20260401-BP-001",
            "inspection_phase": "IN_PROCESS",
            "inspector_id": "INS-001",
            "details": [
                {"inspection_standard_id": "std-001", "measured_value": "12.25"}
            ]
        })

    @task(2)
    def view_dashboard(self):
        """ダッシュボードの閲覧（管理者の操作）。"""
        self.client.get("/api/v1/dashboard/defect-rate?period=DAILY")
```

### 2.2 データ改ざん防止テスト詳細（ST-04）

```
テスト手順:
1. 検査結果を正常に登録する（revision=1）
2. 同じ検査結果を修正する
3. revision が 2 に増加していることを確認
4. 修正前のデータが変更履歴として保持されていることを確認
5. DB の直接 DELETE を試行し、トリガーで拒否されることを確認

期待結果:
- 論理的な削除のみ許可され、物理削除は不可
- 全ての変更に版数（revision）が付与される
- 10年間のデータ保持が可能な設計であること
```
