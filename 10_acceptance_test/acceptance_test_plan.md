## 1. トレーサビリティマトリクス

RFP の各要求事項が、要件定義を経て、テストケースで検証されるまでの対応関係です。

| RFP 要求 | 対応する機能要件 | 受け入れテストケース | カバレッジ |
|---------|----------------|-------------------|-----------|
| 検査結果をタブレットから入力できる | F-02: 検査結果入力 | UAT-01, UAT-02 | 対応済 |
| 検査結果の合否を自動判定できる | F-02: 検査結果入力 | UAT-01, UAT-03 | 対応済 |
| 不良率をリアルタイムに表示するダッシュボード | F-04: 品質ダッシュボード | UAT-05 | 対応済 |
| 品質アラート通知 | F-05: 品質アラート通知 | UAT-06 | 対応済 |
| 検査データを CSV でエクスポート | F-07: 品質レポート出力 | UAT-07 | 対応済 |
| ロット単位のトレーサビリティ | F-06: ロットトレーサビリティ | UAT-08 | 対応済 |
| MES との連携 | F-08: MES連携 | UAT-09 | 対応済 |
| データは 10 年間保存 | NF-10: データ保持期間 | ST-04（システムテスト） | 対応済 |

**カバレッジ: 100%**（RFP の全要求事項がテストケースに対応）

## 2. 業務シナリオテスト

### UAT-01: 検査員による工程内検査（正常系）

| 項目 | 内容 |
|------|------|
| **実施者** | 検査員 |
| **前提条件** | ブレーキパッド製造ライン（LINE-BP）でロット 20260401-BP-001 が製造完了済み |
| **想定所要時間** | 3分 |

**操作手順:**

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | タブレットでシステムにログインする | ログイン画面が表示され、社員番号とパスワードでログインできる |
| 2 | 「検査入力」画面を開く | 検査入力画面が表示される |
| 3 | ロット番号「20260401-BP-001」を選択する | ロット情報（品目名、製造日時、数量）が表示される |
| 4 | 検査工程で「工程内検査」を選択する | 該当品目の検査項目一覧（外径寸法、内径寸法、硬度等）が表示される |
| 5 | 各検査項目の測定値を入力する（例: 外径寸法 = 12.25mm） | 入力欄に数値が入力される |
| 6 | 「検査完了」ボタンを押す | 各項目の合否が自動判定され、結果が画面に表示される |
| 7 | 全項目が「合格」であることを確認する | 全体結果が「合格」と表示され、緑色のマークがつく |

**確認ポイント:**
- [ ] タブレットの画面サイズで問題なく操作できるか
- [ ] 検査基準（上限値・下限値）が画面に表示されるか
- [ ] 合否判定が瞬時に表示されるか（体感 1 秒以内）

### UAT-02: 検査員による工程内検査（不合格ケース）

| 項目 | 内容 |
|------|------|
| **実施者** | 検査員 |
| **前提条件** | ロットが登録済み |
| **想定所要時間** | 5分 |

**操作手順:**

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 検査入力画面でロットを選択する | ロット情報が表示される |
| 2 | 外径寸法に基準範囲外の値を入力する（例: 13.00mm、上限 12.50mm） | 入力欄に数値が入力される |
| 3 | 「検査完了」ボタンを押す | 外径寸法が「不合格」と赤色で表示される。全体結果も「不合格」となる |
| 4 | 不良記録の入力画面が表示される | 不良区分（寸法不良/外観不良/材質不良/その他）の選択肢が表示される |
| 5 | 不良区分「寸法不良」を選択し、不良内容を入力する | テキスト入力欄に不良の詳細を記入できる |
| 6 | 処置（廃棄/手直し/特別採用）を選択する | 選択肢から処置を選べる |
| 7 | 「保存」ボタンを押す | 不良記録が保存され、確認メッセージが表示される |

**確認ポイント:**
- [ ] 不合格の表示が目立つか（赤色表示等）
- [ ] 不良記録の入力が直感的にできるか
- [ ] 入力内容が検査履歴に正しく反映されるか

### UAT-05: 品質管理者によるダッシュボード確認

| 項目 | 内容 |
|------|------|
| **実施者** | 品質管理者 |
| **前提条件** | 当日の検査データが 100 件以上登録済み |
| **想定所要時間** | 5分 |

**操作手順:**

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | PC ブラウザでダッシュボード画面を開く | 本日の品質サマリーが表示される |
| 2 | ライン別不良率のグラフを確認する | 3ライン（BP, DR, ASSY）の不良率が棒グラフで表示される |
| 3 | 期間を「週次」に切り替える | 過去7日間の不良率推移が折れ線グラフで表示される |
| 4 | 特定のラインをクリックする | そのラインの不良区分別パレート図が表示される |
| 5 | 品質アラート一覧を確認する | 未対応のアラートが一覧表示される（発生日時、ライン、不良率） |

**確認ポイント:**
- [ ] グラフが 3 秒以内に描画されるか
- [ ] 5 分間隔で自動更新されるか（画面をリロードしなくても最新データが表示される）
- [ ] 不良率の数値が手動集計と一致するか

### UAT-08: ロットトレーサビリティの確認

| 項目 | 内容 |
|------|------|
| **実施者** | 品質保証担当 |
| **前提条件** | ロット 20260401-BP-001 が全検査工程（受入→工程内→最終→出荷）を完了済み |
| **想定所要時間** | 3分 |

**操作手順:**

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | トレーサビリティ画面を開く | ロット番号の検索欄が表示される |
| 2 | ロット番号「20260401-BP-001」を入力して検索する | ロットの全検査履歴が時系列で表示される |
| 3 | 各検査工程の結果を確認する | 受入検査 → 工程内検査 → 最終検査 → 出荷検査 の順に表示される |
| 4 | 各検査の詳細（測定値、合否、検査員、日時）を確認する | 全項目の測定値と判定結果が表示される |

**確認ポイント:**
- [ ] 顧客から品質データの提出を求められた場合、このデータで対応できるか
- [ ] 変更履歴（修正がある場合）が表示されるか
- [ ] CSV エクスポートで出力できるか

## 3. RFP 課題解決の確認

RFP に記載された 4 つの業務課題が解決されたことを定量的に確認します。

| # | RFP記載の課題 | 解決の確認方法 | 合格基準 | 結果 |
|---|-------------|-------------|---------|------|
| 1 | 検査結果の集計に毎日2時間かかり、リアルタイムでの品質状況把握ができない | ダッシュボードで不良率がリアルタイム表示されることを確認（UAT-05） | 集計作業が不要になり、ダッシュボードで即座に確認可能 | |
| 2 | 不良品の傾向分析ができず、再発防止策の立案が遅れる | 不良区分別パレート図、不良率の推移グラフで傾向分析ができることを確認（UAT-05） | ダッシュボードで傾向が可視化され、その場で分析可能 | |
| 3 | 顧客からの品質データ提出要求に迅速に対応できない | トレーサビリティ画面からロット単位のデータを CSV 出力できることを確認（UAT-08） | ロット番号入力から CSV 出力まで 3 分以内 | |
| 4 | 検査記録の保管が紙ベースのため、過去データの検索に時間がかかる | 検査履歴画面でロット番号・期間・品目等で検索できることを確認 | 検索結果が 3 秒以内に表示される | |
